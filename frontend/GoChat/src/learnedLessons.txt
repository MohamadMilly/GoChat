1 - 
I learned that if i have a map returning items and the list is wrapped via context , i should pass the function mutating the state of the context
provider to the items and also use useCallback hook instead of calling useContext inside each item , so when i change the state of the context provider it will not cause all the items to re-render but only the one that
 is being mutated.
 This way i can optimize the performance of my application and avoid unnecessary re-renders.

 2- 
 Fetching data should be at the top level of the components tree and not inside the child components ,
  so the components that is far from the parent should be dumb components that only receive props and render the UI, 
  this way we can avoid unnecessary re-renders and also we can have a better control over the state of our application.

  3- use exact to invalidate queries with all the keys in the query key only

  4- giving socket object to prisma causes infinite loop

  5- Javascript files are not like React , u should initialize the variables and functions before using them , otherwise you will get an error that the variable is not defined.

  6 - when i need to lift the listener up in the component tree not to lose events when the small low level components are unmounted
      i can use a custom event and emit it to the app using window.dispatchEvent and listen for this event in those low level components for better performance.
